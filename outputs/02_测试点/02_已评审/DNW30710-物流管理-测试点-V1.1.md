## 3. 需求理解

### 3.1 目标说明

- 本需求要解决什么问题：打通物料准备、领料出库、仓储出库、物流任务、收料确认之间的人工物流闭环。
- 面向哪些角色：系统管理员、工厂数据维护人员、计划员、库房管理员、车间操作工、物流执行人、车间/库房主管。
- 主要业务场景：位置主数据维护、位置绑定、计划明细到货位置带值、APS 回写、领料申请带值、空车呼叫、发料周转、工序间周转、入库周转、临时周转、跨组织任务跟踪。

### 3.2 范围说明

- 本次范围：位置主数据维护、工作中心/设备位置绑定、默认物流位置读取、计划与申请单字段贯通、物流任务创建、查询、接收、确认到位、确认完成。
- 非本次范围：自动派工、自动创单、AGV、地图导航、扫码校验、强制将物流任务作为业务前置条件。

## 5. 业务规则测试点

| 编号 | 测试点 | 说明 |
| --- | --- | --- |
| BR-001 | 位置编码唯一 | 新增或编辑位置时编码不可重复。 |
| BR-002 | 已被绑定或被物流任务引用的位置不允许删除 | 删除前必须校验引用关系。 |
| BR-003 | 同一工作中心仅允许存在一条有效绑定 | 重复绑定同一工作中心时应受限。 |
| BR-004 | 同一设备仅允许存在一条有效绑定 | 重复绑定同一设备时应受限。 |
| BR-005 | 到货位置带值优先级正确 | 优先取工序任务关联设备到货位置，其次工作中心。 |
| BR-006 | 无工序信息时取首道工序默认位置 | 当前物料无工序信息时，使用首道工序设备/工作中心位置。 |
| BR-007 | APS 回写始终更新计划明细 | 无论是否已产生申请单，计划明细都更新最新数据。 |
| BR-008 | 仅未出库申请单明细可被 APS 同步更新 | 部分出库、全部出库明细不再更新。 |
| BR-009 | 领料申请允许人工修改到货位置 | 用户可在默认值基础上修改到货位置。 |
| BR-010 | 到货位置选择不受组织权限控制 | 按需求说明可选择所有位置数据。 |
| BR-011 | 发料周转任务仅能在物料已出库后创建 | 待出库或拣配中不得提前创建发料周转。 |
| BR-012 | 空车呼叫无任务明细 | 空车呼叫类任务不应产生物料明细。 |
| BR-013 | 周转类任务必须有物料明细 | 发料/工序间/入库/临时周转均应带物料与数量。 |
| BR-014 | 批次号和序列号继承展示不重复录入 | 来源明细上的批次和序列号在物流任务明细中只读展示。 |

| BR-016 | 收料确认不作为物流任务完成前置 | 已收料但未完成任务时，业务可继续推进。 |
| BR-017 | 物流任务完成不作为收料确认前置 | 任务已完成但尚未收料确认时，应允许存在但提示业务未闭环。 |
| BR-018 | 待接收任务允许直接确认终态 | 待接收任务可直接确认到位或完成，并自动回写接收信息。 |
| BR-019 | 旧业务主流程不被新物流任务能力阻断 | 领料、出库、收料原主流程应继续可用。 |

## 11. 测试数据准备说明

- 前置主数据：组织、位置、工作中心、设备、物料、仓库/库位。
- 前置业务数据：制造订单、物料准备计划、领料出库申请单、出库单。
- 特殊状态数据：未出库、部分出库、全部出库申请单明细；待接收、已接收、已完成、已取消任务；已收料未完成物流、已完成物流未收料等组合状态。
- 权限账号准备：系统管理员、计划员、库房管理员、车间操作工、物流执行人、主管账号。

## 12. 风险与待确认项

- 风险点：APS -> WMS 异步同步链路可能带来时序和重复更新问题。
- 风险点：跨组织任务查看与执行权限边界若定义不清，容易出现可见不可操作或越权处理问题。
- 风险点：历史数据不强制回填，新旧数据并存时页面展示和查询逻辑可能出现空值分支遗漏。
- 风险点：导入和批量引入场景若实现不具备原子性，会直接影响默认位置口径。
- 待确认项：已取消状态的进入条件和页面可见口径是否还需要补充说明。
- 待确认项：空车呼叫和周转类任务在前端是否共用同一入口，以及入口差异化校验细节。
