# 其他问题

本指南汇集了项目实施与运维过程中的高频问题，按**部署运维**、**系统配置**、**安全管理**及**业务操作**等专业维度进行分类。

## 1. 部署运维 (Deployment & O&M)

### 1.1 网络与服务器
*   **Q: 服务器 IP 地址变更后，系统无法访问？**
    *   A: 修改 IP 后，需要同步更新以下配置文件：
        1.  应用服务配置文件 (`application.yml` 或 `.env`) 中的数据库连接地址。
        2.  Nginx 反向代理配置 (`nginx.conf`) 中的 Upstream 地址。
        3.  若使用了注册中心 (如 Nacos/Eureka)，需清除旧的注册实例信息并重启服务。
*   **Q: 部署完成后，PDA 无法连接服务器？**
    *   A: 1. 检查服务器防火墙是否开放了应用端口 (如 8080/80)；2. 确认 PDA 与服务器处于同一网段，或路由策略允许跨网段访问；3. 检查 PDA 的 WIFI 信号强度。

### 1.2 服务管理
*   **Q: Docker 容器启动失败，日志提示 "No space left on device"？**
    *   A: 磁盘空间不足。请清理无用的镜像 (`docker image prune`) 或日志文件。建议为 Docker 数据目录挂载独立的大容量磁盘。
*   **Q: 系统运行一段时间后变慢，重启后恢复？**
    *   A: 可能是内存泄漏或 JVM 堆内存配置过小。建议：1. 检查 JVM 启动参数 (`-Xmx`) 是否合理；2. 开启 GC 日志分析是否存在频繁 Full GC。

## 2. 系统配置 (Configuration)

### 2.1 基础参数
*   **Q: 业务单据号生成重复或乱码？**
    *   A: 请检查 [编码规则](../04-core-modules/master-data/README.md) 配置。
        1.  确认是否在规则中加入了由“年月日+流水号”组成的唯一键。
        2.  在高并发场景下，建议使用 Redis 自增计数器代替数据库计数。
*   **Q: 审批流程走到一半卡住，下一节点处理人收不到待办？**
    *   A: 1. 检查流程设计图中，下一节点是否设置了有效的“候选人组”；2. 确认候选人是否在系统中被禁用了账号。

### 2.2 打印配置
*   **Q: 打印的二维码扫描不出来？**
    *   A: 1. 二维码内容包含中文字符但打印机不支持中文编码（建议转为 Base64 或 UTF-8）；2. 打印浓度过高导致条码粘连；3. 标签纸尺寸设置与实际物理纸张不符。

## 3. 安全管理 (Security)

### 3.1 账号与认证
*   **Q: 为什么新创建的用户提示“账号已锁定”？**
    *   A: 可能是该 IP 地址短时间内触发了暴力破解防护规则。请在安全审计日志中查看该 IP 的请求记录，并在“IP 黑白名单”中进行解封。
*   **Q: 系统是否支持 HTTPS？如何配置？**
    *   A: 支持。建议在 Nginx 层统一配置 SSL 证书，将 443 端口请求转发至后端应用，同时配置 HTTP 强制跳转 HTTPS。

### 3.2 数据安全
*   **Q: 敏感数据（如配方、成本）如何防止泄露？**
    *   A: 1. 在 [角色模型](../03-configuration/角色模型.md) 中严格控制“导出”按钮的权限；2. 开启“水印”功能，在屏幕显示操作员姓名和时间；3. 所有的查询和导出操作均会记录在审计日志中。

## 4. 业务操作 (Business Operations)

### 4.1 生产执行
*   **Q: 报工时提示“当前工序未开工”？**
    *   A: 系统配置了“严格顺序报工”约束。必须完成上一道工序的报工后，才能执行当前工序。如需并行生产，请修改工艺路线属性为“并行”。
*   **Q: 安灯 (Andon) 呼叫后，看板没有弹窗报警？**
    *   A: 1. 检查看板所在终端的网络连接（WebSocket 是否断开）；2. 确认该类型的安灯异常是否配置了“看板通知”的响应策略。

### 4.2 仓储物流
*   **Q: 盘点时发现库存差异，如何平账？**
    *   A: 盘点完成后，系统会自动生成“盘盈单”或“盘亏单”。经财务和库管主管审核通过后，系统会自动调整库存数量，确保存货一致。

## 5. 数据与维护 (Maintenance)

*   **Q: 数据库误删除了数据，能恢复吗？**
    *   A: 依赖于备份策略。如果开启了 Binlog 且有全量备份，可以通过回放 Binlog 恢复到误操作前的时间点。请务必配置每日定时备份。
*   **Q: 历史数据越来越多，系统查询变慢？**
    *   A: 建议启用系统的“数据归档”策略，将超过 3 年的已结案工单和日志迁移至历史数据库（History DB），减轻主库压力。
