# KMMOM 环境配置

## 1. 文档信息

- 文档版本：v1.1
- 更新日期：2026-01-13
- 维护团队：产品研发团队
- 目标读者：实施工程师、运维团队、项目经理

## 2. 文档目标与范围

本文档用于指导 KMMOM 的环境准备与配置，包含：

- 硬件环境：服务器与终端（客户端）配置建议
- 软件环境：基础软件、中间件、应用服务版本基线
- 配置方案：按企业规模、按用户类型（终端角色）的推荐配置

说明：

- 本文档以传统部署模式（物理机/虚拟机）为主；容器化部署方案后续单独补充。
- 本文档中的“推荐配置”为经验建议，实际应结合并发、数据量、网络与客户现状进行评估。

## 4. 硬件环境配置建议

### 4.1 服务器选型建议（信创优先）

- 架构选型：
  - X86 架构（推荐）：优先选用海光（Hygon）处理器服务器，兼容性较好
  - ARM 架构（可选）：选用鲲鹏（Kunpeng）处理器服务器，适合高并发场景
- 推荐品牌：浪潮、华为等国产服务器厂商
- 操作系统：银河麒麟 V10 / 统信 UOS V20

| CPU      | 海光 C86 7000 系列 64 核 × 1   | 鲲鹏 920 64 核 × 1             |
| 内存     | 256GB DDR4                     | 256GB DDR4                     |
| 系统盘   | 2 × 1TB NVMe SSD（RAID1）      | 2 × 1TB NVMe SSD（RAID1）      |

| 用户类型             | 典型场景                                            | 推荐 CPU | 推荐内存 | 推荐存储      | 显示器建议 | 备注                                                     |

## 6. 按企业规模的环境配置方案（服务器侧）

### 6.1 规模划分标准（参考）

| 规模类型 | 车间/分厂 |   员工数 | 并发用户数 | 数据存储容量 | 推荐服务器数量 | 高可用要求                         |
| -------- | --------- | -------: | ---------: | -----------: | -------------: | ---------------------------------- |
| 小型     | 1 个      |     <100 |      20–50 |       ≤500GB |           2 台 | 不需要                             |
| 中型     | 2–3 个    | 100–1000 |    100–500 |         ≤2TB |           4 台 | 数据库主从、中间件集群             |
| 大型     | 5–10 个   |    >1000 |   500–1000 |         ≤5TB |         6–8 台 | 数据库集群、中间件集群、应用多实例 |

并发用户数计算：并发用户数 = 员工总数 × 30%–50%（高峰期可达 60%–80%）。

### 6.2 小型方案（2 台服务器）

适用场景：单一车间/小型分厂，业务流程相对简单，预算有限，优先保证系统可用与性能。

部署约束：

- 2 台物理服务器通常无法实现完整高可用（信创环境对虚拟化支持受限时尤为明显）
- 依赖数据备份与硬件冗余（双电源、RAID）保障数据安全

服务器清单：

| 服务器编号 | 部署内容概述                                        | 说明                     |
| ---------- | --------------------------------------------------- | ------------------------ |
| 服务器 1   | 数据库、中间件（KeyDB、RocketMQ、RustFS）           | 数据层独占，避免资源竞争 |
| 服务器 2   | 中间件（Nacos、SnailJob）、应用服务、前端、负载均衡 | 应用层独占，统一入口     |

### 6.3 中型方案（4 台服务器，高可用）

适用场景：2–3 个车间/分厂，对稳定性有要求，需要高可用保障。

服务器清单：

| 服务器类型          | 数量 | 部署内容概述                                  | 说明                     |
| ------------------- | ---: | --------------------------------------------- | ------------------------ |
| 数据库+中间件服务器 |    2 | 数据库主从、KeyDB、RocketMQ、RustFS、SnailJob | 服务器 1 主、服务器 2 从 |
| 应用服务器          |    2 | Nacos、应用服务、前端、Nginx+Keepalived       | 服务器 3、4 配置一致     |

关键点：

- 数据库主从 + 中间件集群化部署
- 应用双实例部署 + Nginx 主备 + Keepalived（VIP 漂移）

### 6.4 大型方案（7 台服务器，分层部署）

适用场景：5–10 个车间/分厂，数据量大，对性能与高可用要求高。

服务器清单：

| 服务器类型     | 数量 | 部署内容概述                           | 说明                 |
| -------------- | ---: | -------------------------------------- | -------------------- |
| 数据库服务器   |    2 | 达梦数据库 DM8 高可用集群（DM DMP）    | 服务器 1、2 集群部署 |
| 中间件服务器   |    2 | Nacos、KeyDB Cluster、RocketMQ、RustFS | 服务器 3、4 配置一致 |
| 应用服务器     |    2 | 核心业务应用、前端、Nginx+Keepalived   | 服务器 5、6 配置一致 |
| APS/BDS 服务器 |    1 | APS 排程、BDS 跑批、SnailJob           | 服务器 7 独占        |

容器化部署说明：

- 大型企业可评估 Kubernetes 容器化部署以获得自动扩缩容、滚动更新等能力；前提是具备对应运维能力。
