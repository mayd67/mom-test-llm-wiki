# 命令行工具

本文档面向运维人员，提供 MOM 系统常用命令行的简明速查说明。内容按系统基础、容器环境、应用进程和故障排查多个方面进行归类，采用统一的表格形式呈现命令语法和功能说明，便于在安装、升级和日常运维时快速查找。

---

## 1. 系统基础运维

本节涵盖服务器资源监控、网络连通性以及防火墙与端口配置等基础操作。

### 1.1 防火墙与端口

在部署和排查阶段，经常需要确认端口开放情况及防火墙策略。

| 功能             | 示例指令                                     | 说明                            |
| :--------------- | :------------------------------------------- | :------------------------------ |
| 查看防火墙状态   | systemctl status firewalld                   | 查看防火墙服务是否运行          |
| 开放端口（临时） | firewall-cmd --add-port=8080/tcp             | 立即开放 8080 端口，重启后失效  |
| 开放端口（永久） | firewall-cmd --permanent --add-port=8080/tcp | 永久开放端口，需重载后生效      |
| 重载防火墙配置   | firewall-cmd --reload                        | 使永久配置生效                  |
| 查看已开放端口   | firewall-cmd --list-ports                    | 列出当前已开放的端口            |
| 查看端口占用进程 | netstat -tulpn \| grep <port>                | 查看指定端口被哪个进程占用      |
| 查看所有监听端口 | ss -lntp                                     | 查看所有 TCP 监听端口及对应进程 |

### 1.2 资源监控

用于日常巡检服务器 CPU、内存及磁盘等资源情况。

| 功能               | 示例指令                                       | 说明                                            |
| :----------------- | :--------------------------------------------- | :---------------------------------------------- |
| 查看磁盘空间       | df -h                                          | 关注根目录 / 及 Docker 数据目录 /var/lib/docker |
| 查看目录空间占用   | du -sh \* \| sort -hr                          | 在当前目录下按大小排序显示子目录占用            |
| 查看内存使用情况   | free -h                                        | 关注 available 列，评估可用内存                 |
| 查看系统负载与进程 | top                                            | 实时查看 CPU、内存占用及活跃进程                |
| 查找占用较大的文件 | du -ah --max-depth=1 \| sort -hr \| head -n 10 | 查看当前目录下前 10 个占用空间最大的项          |

## 2. 容器环境管理（Docker）

在ARM架构下的部署，MOM 系统的核心服务（数据库、中间件、微服务等）通常以 Docker 容器方式运行。

### 2.1 容器日常管理

| 场景             | 示例指令                         | 说明                         |
| :--------------- | :------------------------------- | :--------------------------- |
| 查看运行中容器   | docker ps                        | 仅查看当前正在运行的容器     |
| 查看所有容器     | docker ps -a                     | 包含已停止的容器             |
| 启动容器         | docker start <name>              | 启动指定容器                 |
| 停止容器         | docker stop <name>               | 停止指定容器                 |
| 重启容器         | docker restart <name>            | 重启指定容器                 |
| 查看资源使用情况 | docker stats --no-stream         | 查看各容器 CPU、内存瞬时占用 |
| 查看容器日志     | docker logs -f --tail 100 <name> | 实时查看最后 100 行日志      |
| 进入容器内部     | docker exec -it <name> bash      | 进入容器 Shell 环境          |

### 2.2 镜像管理与清理

当需要导入新版本镜像或释放磁盘空间时，可使用以下命令：

| 场景           | 示例指令                  | 说明                       |
| :------------- | :------------------------ | :------------------------- |
| 查看本地镜像   | docker images             | 列出本机所有镜像           |
| 导入离线镜像包 | docker load -i images.tar | 从打包文件中导入镜像       |
| 删除指定镜像   | docker rmi <image_id>     | 删除不再使用的镜像         |
| 清理悬空镜像   | docker image prune -f     | 清理未被任何容器使用的镜像 |

### 2.3 Docker Compose 管理

如项目使用 docker-compose.yml 编排多个服务，可参考下列命令：

| 场景               | 示例指令                      | 说明                       |
| :----------------- | :---------------------------- | :------------------------- |
| 启动或更新所有服务 | docker-compose up -d          | 后台启动并在变更时重建服务 |
| 停止并移除所有容器 | docker-compose down           | 停止服务并移除容器         |
| 查看指定服务的日志 | docker-compose logs -f <name> | 实时查看某个服务日志       |

## 3. 应用进程管理（Supervisord）

Supervisord 主要用于管理部署在宿主机上的应用进程，例如消息队列消费者、定时任务和辅助脚本等。

### 3.1 常用控制命令

| 操作           | 示例指令                        | 说明                        |
| :------------- | :------------------------------ | :-------------------------- |
| 查看进程状态   | supervisorctl status            | 查看所有托管进程当前状态    |
| 启动或停止进程 | supervisorctl start/stop <name> | 启停指定进程，支持通配符 \* |
| 重启进程       | supervisorctl restart <name>    | 重启指定进程                |
| 查看进程日志   | supervisorctl tail -f <name>    | 实时查看进程标准输出日志    |

### 3.2 配置变更相关命令

当修改 /etc/supervisord.d 下的配置文件后，可使用下表中的命令完成配置重载和生效：

| 功能     | 示例指令             | 说明                                       |
| :------- | :------------------- | :----------------------------------------- |
| 重载配置 | supervisorctl reread | 重新读取配置文件，识别新增或变更的进程配置 |
| 应用变更 | supervisorctl update | 按配置变更情况重启受影响的进程             |

## 4. 故障排查速查表

遇到系统异常时，可参考下表快速定位问题并选择合适的排查指令。

| 故障现象         | 排查思路                                                              | 推荐指令                                                 |
| :--------------- | :-------------------------------------------------------------------- | :------------------------------------------------------- |
| 服务无法访问     | 1. 检查服务是否启动；2. 检查防火墙端口是否开放；3. 检查端口是否被监听 | docker ps；firewall-cmd --list-ports；telnet <ip> <port> |
| 系统响应卡顿     | 1. 检查 CPU、内存水位；2. 检查磁盘 IO 或空间                          | top；free -h；df -h                                      |
| 数据库连接失败   | 1. 检查数据库容器状态；2. 验证网络连通性                              | docker logs <db_name>；nc -zv <db_ip> <db_port>          |
| 配置修改后未生效 | 1. 确认配置文件是否挂载正确；2. 确认是否执行了重启                    | docker inspect <name>；docker restart <name>             |
