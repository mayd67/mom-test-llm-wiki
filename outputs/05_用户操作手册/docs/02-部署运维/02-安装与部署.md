# KMMOM CLOUD V3.4 安装部署手册

## 1. 文档目的

本文档用于指导实施工程师完成 KMMOM CLOUD V3.4 的标准化安装部署、初始化、验证与问题排查，确保系统可用并满足交付验收要求。

## 2. 适用范围

- 适用角色：实施工程师、运维工程师、交付团队。
- 适用场景：新环境首次安装部署。
- 版本范围：V3.4.x。

## 3. 部署前准备

### 3.1 服务器与网络

- 服务器满足项目硬件资源要求（CPU、内存、磁盘），建议系统盘与数据盘分离。
- 建议提前规划目录：安装包目录（如 `/data`）、应用目录（如 `/opt/kmmom`）、日志目录（如 `/opt/kmmom/logs`）。
- 服务器间网络互通，客户端可访问应用入口。
- 所有服务器时间同步（NTP），避免证书、日志时间错乱导致排障困难。

网络端口（参考 V3.4）：

- `80`
- `1111`
- `8848`
- `40000`
- `40006`
- `40008`
- `40100`

端口要求：

- 以上端口需在服务器本机防火墙、网络防火墙、安全组同时放行。
- 如客户侧端口策略不同，需在部署前完成端口映射表确认。

### 3.2 操作系统与账号准备

- 使用具备安装权限的系统账号执行部署（root 或具备等效 sudo 权限账号）。
- 确认脚本执行权限与换行格式正确（Linux 下为 LF）。
- 关闭或调整会影响安装的安全策略（如 SELinux、最小化防火墙规则），并保留变更记录。
- 安装前确认主机名解析正确（`/etc/hosts` 或 DNS），避免服务间调用失败。

### 3.3 安装介质

- 从制品库获取安装包并核对版本。
- 对安装包执行完整性校验（MD5）。
- 保留原始安装包与校验值截图/记录，便于交付审计。

```bash
md5sum km-mom-release-v3.4.0.zip
```

### 3.4 部署前检查清单

执行安装前，逐项确认：

- 已确认目标版本、部署范围、窗口时间和回退负责人。
- 已完成数据库/中间件账号、密码和连接信息准备。
- 已确认安装目录剩余空间充足（建议至少为安装包大小的 3 倍）。
- 已确认客户网络策略、白名单和访问路径。
- 已完成一次环境快照或备份方案确认（用于异常回退）。

### 3.5 deploy 配置说明（重点）

配置文件位置：

- 主配置：`04-deployment/config/deployment.conf`
- 默认配置：`04-deployment/config/defaults.conf`
- 示例配置：`04-deployment/config/deployment.conf.example`

配置原则：

- 实施时以 `deployment.conf` 为唯一修改入口。
- `defaults.conf` 仅用于查看默认值，不建议直接修改。
- 新环境建议先复制 `deployment.conf.example` 再按目标环境逐项覆盖。

必配参数（部署前必须确认）：

- 节点参数：数据库节点、应用节点、中间件节点 IP/主机名。
- 网络参数：`80`、`1111`、`8848`、`40000`、`40006`、`40008`、`40100` 对应监听与放行策略。
- 数据库参数：地址、端口、库名、用户名、密码。
- Nacos 参数：地址、命名空间、分组、认证信息。
- 存储参数：上传目录、日志目录、临时目录、磁盘挂载点。
- 安全参数：HTTPS 开关、证书路径、密钥路径（启用 HTTPS 时必配）。

建议在项目交付文档中为每个配置项提供以下字段：

- 参数名
- 参数说明
- 是否必填
- 示例值
- 配置文件位置
- 生效阶段（预检/部署/启动）
- 配错现象与排障提示

配置校验动作：

- 修改配置后先执行 `precheck`，确认无阻断项再执行部署。
- 完成部署后校验 Nacos、Nginx、应用入口连通性。
- 建议同时维护“单机版”和“多节点版”两套参数模板，减少现场改配风险。

### 3.6 预检执行

执行预检命令，处理报告中的所有阻断项后再进入安装流程。

```bash
./install.sh --version v3.4.0 precheck
```

预检报告：

- `/opt/kmmom/logs/precheck-report.html`

典型问题：

- 资源不足（CPU/内存/磁盘）
- 端口占用或端口未开通
- 防火墙/SELinux 策略拦截

## 4. 安装部署流程

### 4.1 基础设施安装

```bash
./install.sh --version v3.4.0 install-middleware
```

日志：

- `/opt/kmmom/logs/init-infrastructure.log`

```bash
tail -f /opt/kmmom/logs/init-infrastructure.log
```

### 4.2 产品程序部署

```bash
./install.sh --version v3.4.0 deploy-application
```

日志：

- `/opt/kmmom/logs/deploy-application.log`

```bash
tail -f /opt/kmmom/logs/deploy-application.log
```

说明：

- 审批流相关服务在部分环境可能出现失败提示，需结合整体服务状态判断，不影响主流程时可后续单独处理。

### 4.3 产品初始化

导入产品模型数据：

```bash
./install.sh --version v3.4.0 import-data --types product
```

导入预置数据：

```bash
./install.sh --version v3.4.0 import-data --types preset
```

### 4.4 基础数据与行业数据导入（可选）

导入基础数据：

```bash
./install.sh --version v3.4.0 import-base-data
```

导入行业数据：

```bash
./install.sh --version v3.4.0 import-industry-data
```

日志：

- `/opt/kmmom/logs/import-base-data.log`
- `/opt/kmmom/logs/import-industry-data.log`

## 5. 初始化后配置

按项目要求完成以下配置：

1. 修改超级管理员密码并妥善留档。
2. 配置系统基本信息（客户名称、系统名称等）。
3. 创建三员账号并分配权限（系统管理员、安全保密员、安全审计员）。
4. 配置密码策略、日志审计、权限策略等安全项。

## 6. 服务验证与验收

### 6.1 服务验证

```bash
./install.sh --version v3.4.0 verify-services
```

日志：

- `/opt/kmmom/logs/verify-services.log`

```bash
tail -f /opt/kmmom/logs/verify-services.log
```

验收标准：

- 服务验证检查项全部通过。
- 关键服务进程状态正常。
- 系统入口可访问并可完成登录。

### 6.2 登录验证

示例地址（按实际环境 IP 调整）：

- `http://服务器IP:40000/ant-web`

## 7. 常见故障处理

### 7.1 资源不足

检查资源使用情况并扩容后重试。

```bash
df -h
free -m
```

### 7.2 端口问题

检查端口占用与监听状态：

```bash
netstat -tuln
```

### 7.3 安全策略限制

- 检查防火墙放行规则。
- 检查 SELinux 模式及策略。

### 7.4 中间件或服务启动失败

- 优先查看基础设施日志：`/opt/kmmom/logs/init-infrastructure.log`
- 查看应用部署日志：`/opt/kmmom/logs/deploy-application.log`
- 查看后端服务日志目录：`/opt/kmmom/logs/backend/`

## 8. 标准执行顺序（可直接照此执行）

```bash
./install.sh --version v3.4.0 precheck
./install.sh --version v3.4.0 install-middleware
./install.sh --version v3.4.0 deploy-application
./install.sh --version v3.4.0 import-data --types product
./install.sh --version v3.4.0 import-data --types preset
./install.sh --version v3.4.0 verify-services
```

如需导入基础/行业数据，在 `import-data` 与 `verify-services` 之间增加：

```bash
./install.sh --version v3.4.0 import-base-data
./install.sh --version v3.4.0 import-industry-data
```
