## 3. 用例说明

- 本文档为测试用例初稿，面向测试评审和后续细化执行。
- 用例按“主数据维护、位置绑定、计划回写、任务创建、任务处理、查询展示、异常与回归”组织。
- 本版已吸收测试点评审中识别出的关键缺口：位置主数据、任务状态直达完成、物流任务查询视角、导入失败与原子性。

## 4. 测试用例

### TC-001 位置主数据新增成功

- 前置条件：使用系统管理员账号登录，进入位置管理页面。
- 步骤：
  1. 点击“新增”。
  2. 输入唯一位置编码、位置名称、位置类型、所属组织、备注。
  3. 点击“保存”。
- 预期结果：
  - 保存成功。
  - 列表中出现新增位置。
  - 新位置可被后续绑定关系和物流任务选择。

### TC-002 位置编码唯一性校验

- 前置条件：系统中已存在位置编码 `LOC-001`。
- 步骤：
  1. 在位置管理页面新增位置。
  2. 输入已存在的编码 `LOC-001`，其余必填项填写合法。
  3. 点击“保存”。
- 预期结果：
  - 保存失败。
  - 系统明确提示位置编码重复。
  - 数据不落库。

### TC-003 位置删除成功

- 前置条件：存在一条未被绑定、未被物流任务引用的位置数据。
- 步骤：
  1. 在位置管理列表中找到该位置。
  2. 点击“删除”并确认。
- 预期结果：
  - 删除成功。
  - 列表中该位置消失。

### TC-004 被引用位置禁止删除

- 前置条件：存在一条已被位置绑定关系或物流任务引用的位置数据。
- 步骤：
  1. 在位置管理列表中选择该位置。
  2. 点击“删除”并确认。
- 预期结果：
  - 删除失败。
  - 系统提示该位置已被引用，不允许删除。
  - 原数据保持不变。

### TC-005 位置导入成功

- 前置条件：准备合法的位置导入文件。
- 步骤：
  1. 在位置管理页面点击“导入”。
  2. 上传合法模板文件。
  3. 提交导入。
- 预期结果：
  - 导入成功。
  - 成功结果数量与文件内容一致。
  - 新位置可查询可用。

### TC-006 位置导入单行冲突整批失败

- 前置条件：准备导入文件，其中一行位置编码重复或所属组织非法。
- 步骤：
  1. 在位置管理页面上传该导入文件。
  2. 提交导入。
- 预期结果：
  - 导入失败。
  - 系统明确返回失败原因。
  - 本批数据不应部分成功。

### TC-007 工作中心位置绑定新增成功

- 前置条件：存在可用工作中心、到货位置、出货位置。
- 步骤：
  1. 进入“位置绑定配置-工作中心绑定”页签。
  2. 选择一个未绑定的工作中心。
  3. 填写到货位置、出货位置。
  4. 保存。
- 预期结果：
  - 保存成功。
  - 列表中展示新增绑定。
  - 后续默认位置读取可返回该绑定。

### TC-008 工作中心重复绑定校验

- 前置条件：某工作中心已存在绑定记录。
- 步骤：
  1. 尝试再次为同一工作中心新增绑定。
  2. 提交保存。
- 预期结果：
  - 保存失败。
  - 系统提示同一工作中心只允许一条有效绑定。

### TC-009 设备位置绑定批量编辑成功

- 前置条件：设备绑定列表中至少存在两条记录。
- 步骤：
  1. 切换到“设备绑定”页签。
  2. 勾选多条设备记录。
  3. 点击“批量编辑位置”。
  4. 统一设置到货位置和出货位置并保存。
- 预期结果：
  - 批量保存成功。
  - 所选记录的绑定值被统一更新。

### TC-010 批量引入冲突整批失败

- 前置条件：准备一批包含重复对象或非法对象的批量引入数据。
- 步骤：
  1. 在绑定页面点击“批量引入”。
  2. 选择包含冲突的数据并确认。
- 预期结果：
  - 批量引入失败。
  - 系统提示冲突原因。
  - 不应出现部分成功。

### TC-011 物料准备计划明细默认带出到货位置

- 前置条件：
  - 某工序任务关联设备已绑定到货位置。
  - 制造订单可生成物料准备计划。
- 步骤：
  1. 生成物料准备计划。
  2. 查看物料准备计划明细。
- 预期结果：
  - 明细自动带出到货位置。
  - 带值遵循设备优先规则。

### TC-012 无设备绑定时工作中心兜底带值

- 前置条件：
  - 当前工序关联设备无绑定。
  - 关联工作中心存在到货位置绑定。
- 步骤：
  1. 生成物料准备计划。
  2. 查看对应明细。
- 预期结果：
  - 到货位置使用工作中心绑定值。

### TC-013 无任何绑定时允许为空

- 前置条件：设备和工作中心均未维护绑定。
- 步骤：
  1. 生成物料准备计划。
  2. 查看明细到货位置字段。
- 预期结果：
  - 到货位置为空。
  - 系统不报错，允许后续人工补充。

### TC-014 APS 发布回写计划明细成功

- 前置条件：存在物料准备计划明细，APS 发布结果产生新的 `needTime` 和到货位置。
- 步骤：
  1. 执行 APS 发布。
  2. 查看物料准备计划明细回写结果。
- 预期结果：
  - `needTime` 与到货位置更新为最新值。
  - 回写结果清晰可见。

### TC-015 APS 仅同步未出库申请单明细

- 前置条件：
  - 计划明细已关联多条出库申请单明细。
  - 其中分别存在未出库、部分出库、全部出库状态。
- 步骤：
  1. 执行 APS 发布。
  2. 对比三类状态申请单明细数据。
- 预期结果：
  - 未出库明细同步更新。
  - 部分出库、全部出库明细保持原值不变。

### TC-016 领料申请默认带出并允许修改到货位置

- 前置条件：制造订单齐套分析或制造任务页面已存在需求到料时间和到货位置。
- 步骤：
  1. 发起领料申请。
  2. 检查默认值。
  3. 手工修改到货位置。
  4. 提交申请。
- 预期结果：
  - 默认值正确带出。
  - 用户可手工修改。
  - 提交后申请单明细保存修改后的值。

### TC-017 默认物流位置查询接口设备优先

- 前置条件：设备和工作中心同时存在绑定，且绑定值不同。
- 步骤：
  1. 调用默认物流位置读取接口，传入设备和工作中心。
- 预期结果：
  - 返回设备绑定结果。
  - 返回结果中发出/到达位置口径正确。

### TC-018 默认物流位置查询接口工作中心兜底

- 前置条件：设备无绑定，工作中心存在绑定。
- 步骤：
  1. 调用默认物流位置读取接口。
- 预期结果：
  - 返回工作中心绑定结果。

### TC-019 空车呼叫创建成功

- 前置条件：存在合法到达位置。
- 步骤：
  1. 打开空车呼叫弹窗。
  2. 填写到达位置、可选设备/工作中心、备注。
  3. 提交。
- 预期结果：
  - 创建成功。
  - 返回任务编号与任务编码。
  - 空车呼叫任务无物料明细。

### TC-020 空车呼叫缺少到达位置创建失败

- 前置条件：进入空车呼叫弹窗。
- 步骤：
  1. 不填写到达位置直接提交。
- 预期结果：
  - 创建失败。
  - 系统明确提示到达位置必填。

### TC-021 发料周转任务创建成功

- 前置条件：
  - 出库单已完成出库。
  - 来源明细存在到货位置、物料、数量、批次号/序列号。
- 步骤：
  1. 在出库页面点击“创建物流任务”。
  2. 选择任务类型“发料周转”。
  3. 检查默认带入的发出位置、到达位置、物料明细。
  4. 提交。
- 预期结果：
  - 创建成功。
  - 任务明细包含物料和数量。
  - 批次号、序列号为继承展示。

### TC-022 未出库场景下禁止创建发料周转

- 前置条件：申请单明细仍处于待出库或拣配中。
- 步骤：
  1. 尝试创建发料周转任务。
- 预期结果：
  - 创建失败。
  - 系统提示必须在已出库后才能创建。

### TC-023 临时周转发出位置与到达位置相同校验

- 前置条件：进入临时周转任务创建页面。
- 步骤：
  1. 选择相同的发出位置与到达位置。
  2. 填写物料和数量并提交。
- 预期结果：
  - 创建失败。
  - 系统提示发出位置与到达位置不能相同。

### TC-024 物流任务接收成功

- 前置条件：存在一条待接收物流任务。
- 步骤：
  1. 使用物流执行人账号进入物流任务页面。
  2. 点击“接收任务”。
- 预期结果：
  - 任务状态变为已接收。
  - 接收人和接收时间被记录。

### TC-025 待接收任务直接确认完成

- 前置条件：存在一条待接收的周转类物流任务。
- 步骤：
  1. 不执行接收，直接点击“确认完成”。
- 预期结果：
  - 操作成功。
  - 任务直接进入已完成状态。
  - 系统自动回写接收人和接收时间。

### TC-026 待接收空车呼叫直接确认到位

- 前置条件：存在一条待接收的空车呼叫任务。
- 步骤：
  1. 不执行接收，直接点击“确认到位”。
- 预期结果：
  - 操作成功。
  - 任务进入完成口径的终态。
  - 系统自动回写接收信息。

### TC-027 非法状态下禁止重复处理任务

- 前置条件：某任务已完成或已取消。
- 步骤：
  1. 对该任务再次执行接收、确认到位或确认完成。
- 预期结果：
  - 操作被阻止。
  - 系统提示状态不允许处理。

### TC-028 收料确认与物流任务确认松耦合

- 前置条件：某来源业务同时存在出库单与关联物流任务。
- 步骤：
  1. 先执行收料确认，不处理物流任务。
  2. 再查看任务状态提示。
  3. 反向场景：先完成物流任务，再查看业务收料状态提示。
- 预期结果：
  - 两种动作互不阻断。
  - 系统能展示“已收料未完成物流”或“已完成物流未收料”等提示。

### TC-029 我发起的物流任务查询视角

- 前置条件：当前账号已发起多条不同状态物流任务。
- 步骤：
  1. 进入物流任务页面。
  2. 切换到“我发起的物流任务”视角。
  3. 分别按待接收、已接收、已完成筛选。
- 预期结果：
  - 仅返回当前用户发起的任务。
  - 状态筛选结果正确。

### TC-030 发往本组织的物流任务查询视角

- 前置条件：系统中存在发往当前用户所属组织的物流任务，包括本组织内任务和跨组织流入任务。
- 步骤：
  1. 进入物流任务页面。
  2. 切换到“发往本组织的物流任务”视角。
- 预期结果：
  - 返回到达组织等于当前所属组织的任务。
  - 包括跨组织流入任务。

### TC-031 物流任务管理页多条件查询

- 前置条件：存在多条不同任务编号、类型、状态、组织和来源业务的任务。
- 步骤：
  1. 在物流任务管理页按任务编号、任务类型、状态、发出组织、到达组织、来源业务、发起时间组合查询。
- 预期结果：
  - 查询结果与条件一致。
  - 重置后恢复完整列表。

### TC-032 物流任务列表与详情展示口径

- 前置条件：存在空车呼叫任务和周转类任务各一条。
- 步骤：
  1. 查看物流任务列表。
  2. 打开任务详情。
- 预期结果：
  - 列表展示任务编码、任务类型、状态、发出位置、到达位置、发出组织、到达组织。
  - 周转类任务详情展示来源业务、物料、数量、批次号、序列号。
  - 空车呼叫详情不展示物料明细。

### TC-033 跨组织物流任务处理

- 前置条件：创建一条发出组织与到达组织不同的物流任务。
- 步骤：
  1. 使用发起人查看任务。
  2. 使用目标组织物流执行人查看并处理任务。
  3. 使用主管账号查看任务状态。
- 预期结果：
  - 发起人可追踪自己发起的任务。
  - 目标组织执行人可见并可处理该任务。
  - 主管能统一查看状态。

### TC-034 历史数据兼容性

- 前置条件：存在新增字段上线前的历史申请单、历史出库单、历史任务或历史页面数据。
- 步骤：
  1. 查看历史业务单据列表和详情。
  2. 执行与历史数据相关的原有主流程查询。
- 预期结果：
  - 页面正常展示。
  - 新字段为空时不报错。
  - 原主流程不受影响。
