# **DNW30330****-生产准备**

# 1. **概述**

## 1.1 **原始需求**

在离散制造行业中，生产物料准备是一个关键环节，直接影响生产效率和产品质量。用户希望通过一个符合 ISA-95 L3 标准的 MOM 系统，实现生产物料准备的全流程管理，包括物料准备计划生成、物料齐套检查、领料申请、物料出库、收料确认等功能，解决当前物料准备过程中信息不透明、流程不规范、效率低下的问题。

## 1.2 **需求分析**

**需求原因**：当前物料准备过程依赖人工操作，容易出现信息滞后、物料短缺等问题，影响生产进度。

**信息化系统价值**：通过 MOM 系统，可以实现物料准备过程的自动化和智能化，提高生产效率，减少人为错误。

**行业内友商解决方案**：行业内普遍采用基于微服务架构的 MOM 系统，通过模块化设计实现计划管理、仓储管理、制造执行等功能的协同运作。

#### 2.1.2.6 **库存动态占用与优先级管理**

**角色**：系统自动执行/物料管理员（手动操作）

**活动描述**：根据业务优先级和物料可用性，实现库存的动态占用、分配与释放，并支持优先级调整。关于动态占用的说明：
- 硬占用/锁定：它是一种逻辑上的“库存分配”或“可用性控制”。当对某个物料执行“硬占用/锁定”时，物理库存数量本身不会减少。但是，与该物料相关的“可用库存量”会立即减少。
  - 目的：是为了确保这部分物料被预留给特定高优先级需求，防止其他低优先级需求“抢占”或“误用”这部分物料。
  - 效果：系统在进行物料分配、齐套检查、领料出库等操作时，会将这部分被硬占用的物料视为“不可用”于其他一般需求。这是一种“虚拟锁仓”的概念，它锁住的是物料的“使用权”，而不是物理位置。
  - 举例：仓库里有100个螺丝，其中50个被“硬占用”给了VIP订单A。这时，物理库存仍然是100个，但对于其他订单而言，“可用”的只有50个。当订单A实际领料出库50个后，物理库存才会变为50个。

**输入业务对象**：生产订单、物料需求、库存状态、优先级配置、APS排产结果

**输出业务对象**：物料占用记录、库存状态更新、物料分配报告

**关键业务规则**：

**库存占用规则**：

  1. 不约束（No Constraint）
    - 业务含义：出库时完全忽略已占用数量，只考虑库存数量。
    - 校验逻辑：出库数量 ≤ 库存数量
    - 适用场景：物料非常充裕，或价值极低，无需精细管理占用的场景。
    - 风险：可能导致计划性较差，物料被“无计划”领走。
  2. 软占用（Soft Occupation）
    - 业务含义：出库时需考虑已占用数量。若出库量超过可用库存量（即动用了已占用的物料），系统给出弱提示（警告）。用户可选择强行出库。
    - 校验逻辑：
    - 默认校验：出库数量 ≤ 可用库存量
    - 若出库数量 > 可用库存量，系统提示：“您正在领用已被XX订单（或其他需求）动态占用的物料X件，是否确认强行出库？”
    - 用户选择“是”：允许出库；用户选择“否”：中止出库。
    - 适用场景：对计划有指导性要求，但需要一定灵活性的场景；物料价值较高，但偶尔允许紧急插单打破占用。
  3. 硬占用/锁定（Hard Occupation/Locking）
    - 业务含义：出库时必须严格遵守已占用数量。若出库量超过可用库存量，系统给出强提示（错误），不允许强行出库。
    - 校验逻辑：出库数量 ≤ 可用库存量，否则严格阻止出库，并提示：“物料X已被XX订单（或其他需求）硬性占用，无法出库，请联系相关人员。”
    - 适用场景：高价值物料、关键瓶颈物料、军工等对计划执行严格要求的场景；需要为特定订单进行严格物料预留的场景。

**优先级管理规则**：

- **策略选择**：支持多种优先级策略选择（客户等级/时间紧迫性/排产顺序/产品价值等），当前阶段采用单优先级策略。
  - **策略1：按订单优先级属性动态占用**
  - **业务合理性**：⭐⭐⭐⭐⭐
  - **核心逻辑**：直接反映业务决策和客户重要性
  - **配置方式**：设置客户等级、产品价值、订单紧急程度等属性对应的优先级分数
  - **适用场景**：客户差异化明显的行业，如装配、军工等

  **策略2：按订单开工时间动态占用**
  - **业务合理性**：⭐⭐⭐
  - **核心逻辑**：按时间顺序分配，先到先得
  - **配置方式**：设置不同时间窗口对应的优先级分数
  - **适用场景**：物料充足且同质化程度高的场景，如标准件生产

  **策略3：按排产结果顺序动态占用**
  - **业务合理性**：⭐⭐⭐⭐
  - **核心逻辑**：与生产计划保持一致，确保资源利用效率
  - **配置方式**：设置排产优先级对应的分数区间
  - **适用场景**：排产计划相对稳定的场景，如大批量生产

- **优先级计算**：根据选定的策略计算订单或需求的优先级得分，支持配置化参数设置。
- **物料分配**：按优先级顺序分配物料，高优先级订单/需求优先获得物料预留。
- **冲突处理**：当多个订单/需求竞争同一物料时，按优先级进行智能分配和调整。

**动态占用处理机制(未来扩展)**：

- **事件相关订单触发**：当关键业务事件发生时（如生产订单下达、订单优先级变更、物料入库、物料出库、销售订单变更），立即触发对相关订单的物料占用重新分配。
- **高优先级订单队列主动分配**：当物料库存发生变化（如新物料入库、物料释放）时，系统主动扫描并通知处于"等待"状态的高优先级订单队列，触发其物料的自动分配。
- **定期批处理**：作为兜底机制，定期（例如每小时或每天）对所有物料需求和库存进行全局性的再计算和优化。

**业务规则配置点**：优先级策略配置、优先级参数配置、动态占用触发机制配置、领料申请校验规则配置、齐套占用规则配置。

**业务场景范围**：

#### 2.1.2.12 **生产退料申请**

**角色**：操作工、车间主管、计划员、收料员

**活动描述**：将在生产现场的物料退还给仓库。这是一个综合性的业务流程，覆盖多种退料场景，具体包括：

**工单余料退库**：制造订单或工序完工后，清点出的剩余物料。

**工序过程退料**：生产过程中，因工艺调整等原因，导致已领用的物料不再需要。

**质量退料**：在线边仓或收料环节发现的、未经使用的质量问题物料。

**错发料退料**：在收料环节发现的、由仓库错发的物料。

**订单取消/变更退料**：因订单宏观调整，导致已备或已领的物料需要退库。

**核心概念定义**：

**已领未用物料**：指已通过"收料确认"流程，计入线边仓或工位库存，但尚未在"物料装入"环节被消耗的物料。其可退数量 = 已收料确认数量 - 已装入数量。

**输入业务对象**：制造任务信息、出库单信息、制造订单信息

**输出业务对象**：退料申请单

**关键业务规则**：

退料时机与发起方式根据具体场景而定，系统需在不同功能界面提供针对性的入口。

**业务场景范围**：

|场景 | 场景兼容结论 | 备注|
|--- | --- | ---|
|任务完工后，操作工对剩余的物料进行余料退库 | ⭐本批次 | |
|在收料、生产等环节，对异常物料或计划变更物料进行退库 | ⭐本批次 | |

**流程说明**

计划员接收生产订单和备料清单后，释放生成订单，系统自动生成制造订单和物料准备计划。

计划员根据物料准备计划发起领料申请，创建生产领用出库申请单。

系统将生产领用出库申请单推送给仓储管理员。

仓储管理员根据申请单进行拣配出库操作，生成出库单。

操作工对仓库发货的物料进行收料确认。

如果子件或成品在加工过程中报废，操作工标记子件或成品加工报废。

计划员发起子件或成品的报废入库申请。

系统根据报废申请生成子件或成品的报废申请单。

仓储管理员根据报废申请单进行子件或成品的报废入库操作。

如果成品加工完成，操作工标记成品加工完成。

计划员通过制造订单进行成品完工入库申请。

系统根据成品完工入库申请，生成瑞松成品完工入库申请单。

仓储管理员根据瑞松成品完工入库申请单，对成品进行完工入库操作。

**退料流程：**

操作工在收料确认环节，发现错发料、来料不良等情况，点击"退料申请"按钮，弹出退料申请界面。

用户填写退料数量和原因，点击"确定申请"，系统生成退料申请单。

仓库管理员根据退料申请单，确认退料数量和原因，生成退料确认入库单。

**退料申请**

**点击"报废入库申请"按钮时**：只要选中了订单，即可进入申请界面。

若未选择任何订单点击按钮，则提示："请至少选择一个制造订单进行操作。"

校验规则：无

**输入**：筛选条件

**输出**：制造订单

**处理逻辑**：

填写筛选条件，点击查询，显示查询的制造订单

**验收标准**：

可以根据订单状态筛选制造订单。

点击"合格入库申请"时，能正确校验订单状态并给出提示。

未选择订单点击按钮时，能给出正确的提示。

**验收场景**

**场景1: 合格入库**
- **Given** 制造订单MO001计划生产10件，最后一道工序已完成8件合格品
- **When** 用户进入合格入库界面
- **Then** 系统显示MO001可入库，用户可以对8件合格品进行入库

**场景2: 报废入库**
- **Given** 制造订单MO001在第2道工序报废2件
- **When** 用户进入报废入库界面
- **Then** 系统显示MO001可入库，用户可以对2件报废品进行入库

### **3.12.3 ****完工入库-报废入库申请**

**概述**：对任意状态（如执行中、已完工）的制造订单，支持发起报废入库申请，用于处理生产过程中产生的报废品。

**界面**：界面布局参考"合格入库申请"，但关键字段和逻辑不同。

**界面结构**：

**网格显示列**：制造订单号、物料编码、物料名称、计划数量、已报废数量、本次报废数量、报废原因、计量单位等。

**本次报废数量**：必填，数字编辑框，用户需手动输入。

**报废原因**：非必填，文本输入框

**库房**：必填，默认值读取配置，根据物料类别读取默认废品库房，详见3.1.4.1默认库房配置，未配置或配置错误，则显示空

**交互内容**：用户在查询界面选择一个或多个制造订单，点击"报废入库申请"按钮，弹出报废入库申请界面。

**校验规则**：

本次报废数量 必须为大于0的数字。

本次报废数量 <= (计划数量 - 已申请合格入库数量 - 已申请报废入库数量)。

报废原因 必须选择。

**输入**：

选中的制造订单

本次报废数量

报废原因

**输出**：

类型为"成品报废入库"的入库申请单

**处理逻辑**：

用户在查询界面选择一个或多个制造订单，点击"报废入库申请"按钮，弹出申请界面。

用户填写"本次报废数量"和选择"报废原因"。

点击"确定申请"，系统进行校验。

校验通过后，为每个选中的制造订单创建一条类型为 **"成品报废入库"** 的入库申请单。

更新对应制造订单的"已申请报废入库数量"字段。

申请成功后，关闭弹窗并刷新列表。

**验收标准**：

报废数量超出可报废余量时，系统能正确拦截并提示。

未选择报废原因时，无法提交。

申请成功后，能正确生成"成品报废入库"类型的入库申请单，并更新制造订单的已报废数量。
