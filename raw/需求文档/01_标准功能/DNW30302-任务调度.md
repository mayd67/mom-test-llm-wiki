# DNW30361-制造计划_任务调度

## 1. 概述

### 1.1 原始需求

作为离散制造企业（多品种小批量、机加/装配等）的车间调度员/班组长，我需要一个以“资源（工作中心）”为核心的调度工作台：以周计划表（资源×日期）与资源甘特（占用×时间窗）两种形态呈现工时占用，并支持对工序任务进行二次派工，减少依赖 Excel 的人工维护与反复沟通。

### 1.2 本页定位与边界

- 本页面面向“调度派工 + 周计划执行”，以工作中心为资源建模（设备组/人员组）。
- 排程/调整的最小单元为“工序任务”（Operation Task），遵循前后置约束。
- 工时口径：负荷=计划工时（planHours），用于对比资源可用工时。
- 可用工时口径：按天配置工作中心每日可用工时（workHoursPerDay），不处理班次差异（一期）。
- 二次派工口径：在“工序任务已落到工作中心与计划窗口”之后，将其分配到具体执行对象（设备/人员/班组），形成可执行的派工单元，并进入开工/报工闭环。
- 本页不替代“整体计划/订单甘特”视图；订单维度的交期、版本、变更等能力复用既有设计。

## 3. 页面信息架构

### 3.1 页面布局（推荐）

- 顶栏：筛选与功能区。
- 中间：资源周计划表 / 资源甘特（主视图二选一）。
- 右侧：任务调整与派工面板（侧滑）。

### 3.2 顶栏

- 筛选框支持布局调整，以适配不同页面宽度与使用习惯。
- 当前暂无特殊业务功能。
- 后续可按“任务管理”的功能布局扩展。

### 3.3 主视图A：资源周计划表（工作中心×日期）

用于替代 Excel 周计划表的核心视图，重点展示“天粒度的工时安排与超负荷日期”。

- 行：工作中心。
- 列：日期（天）。
- 单元格内容：
  - 当天工序任务列表（默认按计划开始排序；未排程不显示）。
  - 当日汇总：计划工时合计 / 可用工时 / 差值（超负荷突出显示）。
- 交互：
  - 工序任务在同一工作中心内跨日期拖动（天粒度），系统自动调整计划窗（默认保持工时不变）。
  - 通过面板变更工作中心/计划窗/计划工时，并即时刷新当日汇总与超负荷提示。

### 3.4 主视图B：资源甘特（工作中心×时间轴）

用于检查日内冲突、并行能力不足与具体时间窗安排。

- 行：工作中心。
- 列：时间轴（默认天，支持小时）。
- 条：工序任务计划窗（planStart→planEnd）；
- 冲突提示：
  - 小时刻度：按时间窗重叠结合并行能力提示冲突。
  - 天刻度：以超载提示为主（红线/热力/标记均可）。
- 编辑方式：
  - 点击任务条打开侧滑面板编辑（计划窗、工作中心、计划工时、锁定）。

### 3.5 右侧：任务调整与二次派工面板

- 任务调整：
  - 编辑计划开始/结束、计划工时、工作中心、锁定标识、备注。
  - 显示该工作中心在当前视窗/本周的工时占用与超负荷提示（统计方式与周计划表一致）。
- 二次派工：
  - 选择执行对象（设备/人员/班组），支持多人/多设备派工（拆分派工）。
  - 可选：细化派工时间窗（默认继承工序任务计划窗）。
  - 生成派工任务并跳转/联动到派工执行列表（开工/报工）。

## 4. 业务规则与校验

### 4.1 周计划查看（天粒度）

- 当日计划工时合计：汇总该天内的工序任务 planHours（未完成优先）。
- 当日可用工时：workHoursPerDay × capacityUnits。
- 超负荷判定：当日计划工时合计 > 当日可用工时，标记超负荷并提示主要影响任务。

### 4.2 冲突判定（小时粒度）

- 默认同一工作中心同一时间窗不允许任务条重叠。
- 若工作中心配置并行能力（capacityUnits>1），则允许在能力范围内重叠，超出则提示冲突。

### 4.3 前后置与调整传播

- 若工序之间存在前后置关系：后工序计划开始不得早于前工序计划结束。
- 调整导致前后置冲突时，系统提供“一键顺延后续工序”以恢复可行排程；用户可取消本次调整。

### 4.4 二次派工规则

- 派工前置：仅允许对状态为“已排程/进行中”的工序任务创建派工任务；未排程需先补齐计划窗。
- 执行对象归属：执行对象需属于工序任务 workCenterId（跨工作中心派工弱校验提示）。
