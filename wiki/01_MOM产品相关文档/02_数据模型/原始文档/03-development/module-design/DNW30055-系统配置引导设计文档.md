# 配置引导中心功能设计文档

## 1. 需求背景

### 1.1 原始需求
- **需求来源**：DNW30055 系统配置引导
- **项目阶段**：系统初始化与基础数据导入阶段
- **整体流程**：任务准备 → 基础设施安装 → 产品程序部署与配置 → 产品初始化 → **系统初始化引导** → **基础数据导入** → 功能演示 → 验收与交付

### 1.2 业务背景
系统安装完成后，预定义数据已导入（包含：配置管理、超级管理员、编码规则、菜单），系统可通过超级管理员登录。但需要进行系统级开关配置和基础数据导入才能正常运行。

**当前痛点**：
- 系统级开关配置（三员管理、预置角色、安全策略等）缺乏首次引导，用户不知道这些配置的存在和影响
- 需要进入多个页面维护数据，查找对应页面耗时
- 配置项分散，缺乏统一入口
- 配置顺序依赖关系不清晰
- 必选配置与可选配置不清晰，容易遗漏关键配置

### 1.3 业务目标
- **首次引导**：系统部署后首次登录，通过向导式引导帮助超级管理员快速完成系统级开关配置
- **效率提升**：通过统一的配置引导中心，快速完成基础数据导入
- **质量保障**：明确配置顺序和依赖关系，确保关键数据不遗漏
- **量化目标**：3分钟内完成初始化引导，15分钟内完成配置中心主线配置（含数据准备时间）

### 1.4 业务场景
系统安装完成后，预定义数据导入后，需要完成以下配置：
1. **系统初始化引导**（首次登录时）：三员管理启用、预置角色启用、系统外观配置、安全策略快捷配置
2. **配置中心配置**（通过配置中心入口）：
   - 角色及角色拥有的菜单权限、数据权限
   - 组织与用户（行政组织、工厂组织、用户）
   - 工厂资源（供应商、设备定义、工装定义、工作中心、库房库位）
   - 产品与工艺（物料信息、MBom、工序库、工艺路线）

## 2. 功能设计

### 2.1 功能一：系统初始化引导

#### 2.1.1 功能概述
系统部署后超级管理员首次登录时，自动弹出初始化引导，帮助用户快速完成系统级开关配置。引导采用两段式设计：先通过弹窗向导完成配置，再通过气泡引导帮助用户认识系统界面。

#### 2.1.2 触发条件与生命周期

| 项目 | 说明 |
|------|------|
| 触发角色 | 仅超级管理员（superAdmin） |
| 触发条件 | 系统部署后超级管理员首次登录，且`SYS_CONFIG_DEFINITION`表中配置编码为`SYSTEM_INIT_GUIDE_COMPLETED`的配置项的默认值为`false`或不存在 |
| 触发次数 | 仅触发一次，完成后不再弹出 |
| 状态存储 | 在`SYS_CONFIG_DEFINITION`表中添加配置项：<br>- 配置编码：`SYSTEM_INIT_GUIDE_COMPLETED`<br>- 配置名称：系统初始化引导完成状态<br>- 默认值：`false`（表示未完成）<br>- 引导完成后：更新默认值为`true`，并将软删除标记（`CSOFT_DELETE_FLAG`）设置为`1`（表示已删除/已完成） |
| 重新进入 | 不提供重新进入引导的入口，用户去配置中心修改 |

#### 2.1.3 第一阶段：弹窗向导

**交互形式**：
- 居中弹窗 + 顶部步骤条
- 弹窗内完成所有配置（纯引导式），不跳转到其他页面
- 每个步骤提供「跳过」和「下一步」按钮
- 每步可跳过，但必须走完所有步骤才能完成引导
- 弹窗支持滚动，以容纳内容较多的步骤（如系统外观配置）
- 弹窗右上角提供关闭按钮，点击关闭则丢弃所有配置，下次登录重新引导

**数据暂存机制**：
- **前端暂存**：引导过程中所有配置项的数据都暂存在前端，**不会立即保存到数据库**
- **一次性提交**：只有点击「完成」按钮时，才一次性提交所有配置到后端
- **全部成功或全部失败**：使用`@Transactional`确保所有配置写入在同一事务中，如果任何配置写入失败，整个事务回滚，所有配置都不保存
- **中断处理**：如果用户关闭弹窗中断引导，所有暂存的配置数据都丢弃，下次登录重新开始引导

**步骤1：三员管理启用**

- **配置项**：是否启用三员分权管理（开关，默认关闭）
- **说明**：启用后系统将采用超级管理员、安全管理员、系统管理员三种角色的分权管理模式

**步骤2：预置角色启用**

- **配置项**：是否启用预置角色（开关 + 角色列表勾选，默认关闭）
- **预置角色列表**：系统管理员、厂际协同计划员、生产厂长、生产主管、工艺主管、质量主管、设备主管、工装管理员、仓库主管、库管员、工时管理员、班组长、操作工、质检员、设备保养/维修专员、工装检定/维保专员（共16个角色，默认全选）
- **说明**：启用后系统自动创建对应角色及其默认权限配置

**步骤3：系统外观配置**

- **系统标识配置**：平台Logo、网站图标、用户头像、系统主标题、系统子标题
- **登录页配置**：登录页背景
- **首页配置**：首页背景、欢迎语
- **系统水印设置**：是否开启水印、水印类型（内容水印/图片水印）、水印内容/文件、水印附加信息（当前用户名、当前时间、当前组织）
- **说明**：跳过此步骤时，所有配置项使用系统默认值

**步骤4：安全策略快捷配置**
- **密码策略**：密码最小长度、复杂度要求（大小写、数字、特殊字符）、过期天数
- **空闲锁定**：是否启用空闲锁定，空闲时长（分钟）
- **多IP登录控制**：是否允许同一账号多地点同时登录
- **登录失败锁定**：连续登录失败次数上限、锁定时长（分钟）
- **说明**：跳过此步骤时，所有配置项使用系统默认值

**完成动作**：
1. 点击「完成」按钮
2. **一次性提交所有配置**（跳过的步骤使用系统默认值）：
   - 前端将所有暂存的配置数据一次性提交到后端
   - 使用`@Transactional`确保所有配置写入在同一事务中
   - **全部成功或全部失败**：如果任何配置写入失败，整个事务回滚，所有配置都不保存，引导状态不更新
   - 如果提交成功，所有配置同时生效
3. 关闭弹窗，进入系统主界面
4. 自动触发第二阶段气泡引导
5. 标记系统初始化引导已完成：
   - 更新`SYS_CONFIG_DEFINITION`表中配置编码为`SYSTEM_INIT_GUIDE_COMPLETED`的配置项
   - 将默认值（`CDEFAULT_VALUE`）更新为`true`
   - 将软删除标记（`CSOFT_DELETE_FLAG`）设置为`1`（表示已删除/已完成）
   - 后续登录不再触发引导

**重要说明**：
- 初始化引导对应的只有一个，最终完成一起提交
- 引导过程中的配置都暂存在前端，不会立即保存
- 只有点击「完成」按钮时，才一次性提交所有配置
- 如果有报错，则全部不成功，所有配置都不保存，用户可重新进入引导

#### 2.1.4 第二阶段：气泡引导

弹窗向导完成后，在系统主界面上以气泡形式指引用户认识关键入口位置。

**气泡引导步骤**：
1. **配置中心入口**：指向右上角用户头像下拉菜单 → 配置中心
2. **功能导航**：指向左侧导航菜单区域
3. **消息与待办**：指向顶部消息中心/待办区域

**交互规则**：
- 气泡展示时，指向目标位置，背景半透明遮罩，目标区域高亮
- 点击「下一个」进入下一步气泡引导
- 点击「跳过」或关闭按钮，整个气泡引导结束，不再出现
- 走完所有步骤自动结束，不再出现
- 气泡引导状态不持久化，关闭即结束

### 2.2 功能二：基础数据导入

#### 2.2.1 配置项清单分组说明（仅展示）

根据DNW30055需求，配置中心页面首先提供**配置项清单区域**，按业务类别共分为**5个配置组**，仅用于展示和跳转，不涉及Excel导入能力（导入能力在后文“导入分组”中单独说明）：

| 序号 | 配置组名称 | 必选标识 | 说明 |
|------|-----------|---------|------|

| 2 | 安全配置 | ★ | 包含：安全配置、角色管理、角色授权、权限查询【TBD后面开发】 |

| 4 | 产品与工艺 |  | 包含：物料信息、MBOM、工艺路线 |
| 5 | 个性化配置 |  | 包含：外观配置、界面布局配置、审批流配置、业务参数配置、文档工具、文档模型工具配置 |

**说明**：

- **配置项清单区域**：用于展示配置项列表并跳转到对应功能页面，不在此处区分是否支持Excel导入
- **导入能力**：Excel批量导入能力通过后文的“导入分组”单独定义，与本表解耦
- **必选配置**：标注★的配置组为必选配置，系统正常运行所必须完成的配置项

#### 2.2.3 角色权限处理

**重要说明**：角色及角色拥有的菜单权限、数据权限是系统安装完成后**必须配置**的内容，但**不通过Excel导入**，而是通过以下方式处理：

**处理方式**：
1. **预置角色定义和菜单权限**：通过系统初始化引导或初始化JSON脚本导入
   - 在系统初始化引导的"步骤2：预置角色启用"中，用户可选择启用哪些预置角色
   - 启用后系统自动创建对应角色及其默认权限配置（权限定义详见「导航按钮角色权限默认配置」文档）
   - 包含角色定义、角色与菜单的关联关系
2. **数据权限**：默认设置为"本部门及以下"
   - 在角色初始化时自动设置
   - 后续可在配置引导中心通过界面调整
3. **角色管理**：在配置引导中心通过界面配置
   - 提供"角色管理"链接，点击后跳转到角色管理页面
   - 可创建和管理系统角色
4. **角色授权**：在配置引导中心通过界面配置
   - 提供"角色授权"链接，点击后跳转到角色授权页面
   - 为角色分配功能权限，将角色关联到用户，使用户获得权限

**为什么不用Excel导入**：
- 角色和菜单权限结构复杂，涉及多对多关系
- 初始化时已有完整的角色权限配置模板
- 通过JSON初始化更灵活，便于版本管理和维护

|------------|------------|------|

- 主表和子表都在同一个Excel文件的不同Sheet页中
- 导入时需要识别主表和子表的关联关系
- 工艺路线工序依赖的工序库数据在第二组（工厂资源）中导入

### 2.5 功能定位

本方案包含两个核心功能：

**功能一：系统初始化引导**
- 系统部署后超级管理员首次登录时自动弹出
- 通过弹窗向导完成系统级开关配置（三员管理、预置角色、系统外观、安全策略）
- 通过气泡引导帮助用户认识系统界面关键入口
- 仅触发一次，完成后不再弹出

**功能二：配置中心**
- 提供分类别、可视化的配置引导界面，其中包含**配置项清单区域**（仅展示配置项并提供跳转）
- 提供**批量导入区域**，支持Excel批量导入基础数据（3个导入分组：系统配置、工厂资源、产品与工艺）
- 批量导入区域中，每个分组一个Excel文件，包含多个Sheet页（主表和子表）
- 批量导入区域支持模板下载和批量导入
- **导入前需选择**：
  - **导入模式**：跳过模式（默认）或覆盖模式
  - **创建人**（仅导入非用户数据时选择）：
    - 首次导入用户数据时：创建人自动设置为用户自身，无需选择
    - 导入其他数据（物料、设备、工艺路线等）时：需选择创建人（从用户列表中选择，建议选择数据管理员或超级管理员）
- **错误处理**：错误数据跳过，写入Excel错误信息列，支持部分成功和失败
- **重复处理**：根据导入模式处理（跳过或覆盖），在Excel错误信息列标注
- 导入和查看是分开显示的，导入分组仅说明支持导入的组
- 提供**配置完成度检查**功能，显示已完成和未完成的配置项，必选配置必须完成

## 3. 程序设计

### 3.1 代码位置

#### 3.1.1 系统初始化引导代码位置

**模块位置**：统一放在 `km-mom-platform-biz-config` 模块

**设计说明**：
- 系统初始化引导统一为一个接口，由`ConfigAppService`统一处理
- 参照现有的开启三员（`enableSystemThree()`）和配置修改（`saveConfigValues()`）接口的参数定义方式
- 所有步骤的配置数据通过一个DTO统一传入，一次性提交处理

**后端包路径结构**（待实现）：
```
com.kmsoft.mom.platform.biz.config
├── application
│   └── ConfigAppService.java                           # 配置应用服务（统一处理初始化引导）
│       + completeInitGuide(cmd: InitGuideCompleteDTO): Result<Void>
│         完成系统初始化引导，一次性提交所有配置
│           
├── model
│   └── dto
│       └── configguide
│           └── InitGuideCompleteDTO.java               # 初始化引导完成DTO
│               + enableThreeSystem: Boolean            # 是否启用三员管理
│               + presetRoles: List<String>              # 启用的预置角色编码列表
│               + appearanceConfig: AppearanceConfigDTO # 系统外观配置
│               + securityPolicy: SecurityPolicyDTO     # 安全策略配置
│
└── remote
    └── ConfigController.java                            # 配置控制器（复用现有）
        + completeInitGuide(@RequestBody Request<InitGuideCompleteDTO> request)
          完成系统初始化引导接口
```

**接口设计**：
- **接口路径**：`POST /config/completeInitGuide`
- **参数**：`Request<InitGuideCompleteDTO>`
- **返回**：`Response<Void>`
- **说明**：统一处理所有步骤的配置，一次性提交，全部成功或全部失败

### 3.4 导入器处理逻辑说明

#### 3.4.1 FactoryResourceDataImporter（工厂资源数据导入器）

**处理逻辑**：

1. **extract() - 数据提取**
   - 解析Excel文件，识别9个Sheet页：工厂组织、供应商、设备定义、工装定义、工作中心、库房、库位、资质等级、工序库
   - 将每个Sheet页的数据转换为`SheetData`对象

2. **transform() - 数据转换**
   - 遍历所有Sheet页，根据Sheet名称匹配实体类型
   - 对每个Sheet调用`BaseTransformStrategy.transform()`转换
   - 合并所有Sheet的转换结果

3. **validate() - 数据校验**
   - 对每个Sheet调用`BaseValidateStrategy.validate()`进行基础校验
   - 处理库位的依赖校验：
     - 收集所有库位引用的库房编码
     - 检查这些库房是否在当前导入的库房Sheet中存在
     - 如果不存在，记录错误信息
   - 合并所有Sheet的校验结果和错误信息

4. **save() - 数据保存**
   - 按依赖顺序排序Sheet页：工厂组织 → 供应商、设备、工装、工作中心、资质等级、工序库 → 库房 → 库位
   - 依次调用`ImportSaveService.save()`保存每个Sheet的数据
   - 处理库位对库房的关联关系（通过库房编码匹配）
   - 如果任何一个Sheet保存失败，整个事务回滚

---

#### 3.4.2 ProductProcessDataImporter（产品与工艺数据导入器）

**处理逻辑**：

1. **extract() - 数据提取**
   - 解析Excel文件，识别7个Sheet页：物料、MBom、MBomNode、工艺路线、工艺路线工序、工艺路线工序物料、工艺路线工步
   - 将每个Sheet页的数据转换为`SheetData`对象

2. **transform() - 数据转换**
   - 按依赖顺序处理各个Sheet页：
     - **物料Sheet**：调用`ExcelDataImporter`的transform方法（使用`BaseTransformStrategy`）
     - **MBom Sheet**：调用`BomDataImporter`的transform方法（使用`BomTransformStrategy`）
     - **MBomNode Sheet**：作为MBom的子表，在`BomTransformStrategy`中处理
     - **工艺路线Sheet**：调用`RoutingDataImporter`的transform方法（使用`RoutingTransformStrategy`）
     - **工艺路线工序Sheet**：作为工艺路线的子表，在`RoutingTransformStrategy`中处理
     - **工艺路线工序物料Sheet**：作为工艺路线的子表，在`RoutingTransformStrategy`中处理
     - **工艺路线工步Sheet**：作为工艺路线的子表，在`RoutingTransformStrategy`中处理
   - 合并所有Sheet的转换结果

3. **validate() - 数据校验**
   - 按依赖顺序校验各个Sheet页：
     - **物料Sheet**：调用`ExcelDataImporter`的validate方法（使用`BaseValidateStrategy`）
     - **MBom Sheet**：调用`BomDataImporter`的validate方法（使用`BomValidateStrategy`）
       - 校验MBom依赖的物料是否存在
     - **MBomNode Sheet**：在`BomValidateStrategy`中校验
       - 校验MBomNode依赖的MBom主表和物料是否存在
     - **工艺路线Sheet**：调用`RoutingDataImporter`的validate方法（使用`RoutingValidateStrategy`）
       - 校验工艺路线依赖的物料是否存在
       - 校验工艺路线工序依赖的工序库是否存在（工序库在第二组工厂资源中导入）
     - **工艺路线工序Sheet**：在`RoutingValidateStrategy`中校验
       - 校验工艺路线工序依赖的工艺路线主表和工序库是否存在
     - **工艺路线工序物料Sheet**：在`RoutingValidateStrategy`中校验
       - 校验工艺路线工序物料依赖的工艺路线主表和物料是否存在
     - **工艺路线工步Sheet**：在`RoutingValidateStrategy`中校验
       - 校验工艺路线工步依赖的工艺路线主表是否存在
   - 合并所有Sheet的校验结果和错误信息

4. **save() - 数据保存**
   - 按依赖顺序保存各个Sheet页：
     - **物料Sheet**：调用`ExcelDataImporter`的save方法（使用`ImportSaveService`）
     - **MBom Sheet**：调用`BomDataImporter`的save方法（使用`ImportSaveService`）
     - **MBomNode Sheet**：在`BomDataImporter`的save方法中一起保存
     - **工艺路线Sheet**：调用`RoutingDataImporter`的save方法（使用`ImportRoutingSaveService`）
     - **工艺路线工序Sheet**：在`RoutingDataImporter`的save方法中一起保存
     - **工艺路线工序物料Sheet**：在`RoutingDataImporter`的save方法中一起保存
     - **工艺路线工步Sheet**：在`RoutingDataImporter`的save方法中一起保存
   - 如果任何一个Sheet保存失败，整个事务回滚

**关键点**：
- 复用现有的导入器（`BomDataImporter`、`RoutingDataImporter`、`ExcelDataImporter`）
- 按依赖顺序处理各个Sheet页
- 主表和子表在对应的导入器中一起处理
- 工艺路线工序依赖的工序库在第二组（工厂资源）中导入，需要在校验时检查工序库是否存在
- 所有Sheet页的错误信息都会合并，最终返回包含所有Sheet页的完整Excel文件
- 最终调用基座的`AbstractDataImporter`的`importData`方法，基座负责extract、transform、validate、save的流程控制
