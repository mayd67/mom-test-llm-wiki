# 测试用例模板说明

本文档定义当前知识库中测试用例输出到 MeterSphere 的标准模板口径。

模板来源依据：

- `wiki/01_MOM产品相关文档/04_测试用例/02_测试用例样例/`
- `wiki/01_MOM产品相关文档/04_测试用例/03_测试用例导入模板/`

## 一、输出字段

MeterSphere 导入模板字段统一为：

| 字段 | 说明 |
| --- | --- |
| 用例名称 | 测试用例标题 |
| 所属模块 | MeterSphere 中的模块路径 |
| 标签 | 需求编号+需求名称 |
| 前置条件 | 执行前需要准备的环境、数据、状态 |
| 步骤描述 | 测试步骤 |
| 预期结果 | 步骤对应预期 |
| 编辑模式 | 固定使用 `TEXT` |
| 备注 | 选填 |
| 用例状态 | 默认 `未开始` |
| 责任人 | 默认留空，后续按需补充 |
| 用例等级 | 根据规则填写 `P0 / P1 / P2` |

## 二、固定默认值

### 1. 标签字段

标签统一取：

`需求编号-需求名称`

示例：

- `DNW30120-工序库`
- `DNW30710-物流管理`
- `DNW31500-智能问数`

### 2. 编辑模式

统一固定为：

`TEXT`

说明：

- 当前模板按文本型步骤输出
- 不使用 `STEP` 模式

### 3. 用例状态

统一默认值：

`未开始`

### 4. 责任人

统一默认值：

留空

说明：

- 不在模板中预填责任人
- 后续按项目、迭代或人员分工补充

## 三、用例等级规则

### P0

适用场景：

- 主流程核心功能
- 提交、保存、审批、发布、出库、收料等关键闭环
- 影响业务是否可继续推进的能力
- 一旦失败会直接阻断上线或主要业务流程

典型例子：

- 保存失败
- 提交流程无法完成
- 关键状态流转错误
- 主单/明细核心联动错误

### P1

适用场景：

- 重要业务规则
- 高频使用页面功能
- 查询、编辑、列表展示、常见异常处理
- 影响范围较大但不一定阻断主流程

典型例子：

- 查询结果不正确
- 规则校验缺失
- 权限控制异常
- 常用联动字段未正确回写

### P2

适用场景：

- 辅助功能
- 低频场景
- 提示文案
- 边界兼容性
- 一般异常分支

典型例子：

- 备注字段展示异常
- 低频筛选场景
- 非关键提示文案
- 个别边界值处理

## 四、推荐命名规则

### 1. 用例名称

建议格式：

`功能点 + 场景 + 预期`

示例：

- `物流任务列表按状态筛选成功`
- `待接收任务可直接确认完成`
- `被引用位置删除失败`

### 2. 所属模块

建议使用 MeterSphere 模块树全路径，例如：

`/全部用例/MOM产品功能用例/03-仓储物流/物流管理/物流任务/列表`

## 五、步骤编写要求

由于统一使用 `TEXT` 模式，步骤描述建议按自然语言编号书写：

```text
1. 进入物流任务页面
2. 选择“我发起的物流任务”视角
3. 选择状态为“待接收”
4. 点击查询
```

预期结果也建议按对应编号书写：

```text
1. 页面查询成功
2. 仅展示当前用户发起的任务
3. 列表中的任务状态均为“待接收”
```

## 六、适用方式

后续从知识库生成测试用例时，统一遵循以下口径：

- 需求文档从 `wiki/01_产品需求文档/` 读取
- 设计和数据模型从 `wiki/02_数据模型/` 读取
- 测试点从 `outputs/02_测试点/02_已评审/` 读取
- 输出到 MeterSphere 的用例文件统一使用本模板结构
