# KMMOM 加密卡

本文面向运维人员，提供KMMOM 加密卡在 Linux 系统上的部署配置说明。

## 简介

KMMOM 加密卡用于提供正式许可校验与授权服务，保障系统合规、安全、稳定运行；支持“服务端 + 客户端”部署形态：客户端必须与 KMMOM 系统同机部署，服务端可同机或独立服务器部署。本文覆盖安装包获取、服务端/客户端部署、Nacos 参数配置与容器挂载要点，提供联通校验与常见问题指引，帮助完成安装、启用与生效验证。
- 在线用户控制：限制并发在线用户数，不限制登录后功能使用（功能使用由权限体系控制）。
- 时间期限控制：按购买时约定的许可有效期进行校验，到期后授权校验不通过，需联系开目客户服务中心续期并更换许可。

## 加密卡部署配置

### 1. 部署配置说明

- 联系开目客户服务中心（027-67869871,4008860255），获取 **匹配当前服务器的加密卡程序安装包及相关安装说明文档**（提前使用命令“uname -a”查看当前服务器的CPU架构和操作系统信息）；
- 加密卡分服务端和客户端，客户端 **必须** 与 KMMOM 系统部署在同一台服务器，服务端可 **同机或独立** 服务器部署；
- 在服务端部署时，需要注意先在服务器获取申请码，将得到的申请码文件license.txt提交给开目客服,获取后缀为lic的开目允可文件。

### 2. 服务端部署

- 具体部署详情见从开目客户服务中心获取到的安装说明文档，操作到启动服务即可。
- 通过申请码获取到的文件包中，**仅需要** 后缀为lic文件。

> **注意:**
>
> - 服务端默认通信端口为5600，请在服务器与安全设备上放行。
> - 确认安装包版本与目标环境（CPU/OS）一致。

### 3. 客户端部署及Nacos 配置

- **客户端部署**：将KmEncryptCardClient的jar包放置在 KMMOM 所在服务器目录下，例如：/data/KmPermitClient/。
- **Nacos 配置**：
  - 登录Nacos 管理页面，选择配置列表，编辑 **km-mom-gateway.yml** 配置文件，在 **kmmom** 下增加以下加密卡配置内容（根据实际情况填写 **加密卡服务IP[serverIp]**、**加密卡服务端口[serverPort]**、**KmPermitClient的jar包放置路径[dpToolDir]** 参数）：
    ```
     # 加密卡配置
    encryption:
      serverPort: 5600
      serverIp: "127.0.0.1"
      dpToolDir: "/data/KmPermitClient"
    ```
  - 配置完成后，点击“发布”，创建/重启 **km-mom-gateway网关服务** 生效，否则无法正常使用 KMMOM 系统。
