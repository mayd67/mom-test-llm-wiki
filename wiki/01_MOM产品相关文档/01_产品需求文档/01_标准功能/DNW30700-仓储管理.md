# **DNW30700-仓储管理**

# 1. **概述**

### 1.1.1 **原始需求**

在离散制造行业中，尤其是机加/装配领域，企业面临着多品种小批量的生产模式挑战。以某离散制造企业为例，其在仓储管理环节存在以下原始需求和问题痛点：

**库存追溯困难**：在生产过程中，当产品出现质量问题时，无法快速追溯到所使用原材料的批次和供应商，导致问题排查时间长，影响客户交付和企业信誉。

**物料配送不及时**：生产线上经常因为物料未能及时配送到位而出现停工待料的情况，严重影响生产效率和设备利用率。

**库存积压严重**：由于缺乏精准的库存管理和预测，企业存在大量原材料和半成品的积压，占用了大量资金和仓库空间，增加了库存成本。

**仓储布局不合理**：仓库内物料存放混乱，不同种类和使用频率的物料未进行合理分区，导致物料查找和搬运时间长，降低了仓库作业效率。

通过信息化系统（MOM 系统中的仓储管理模块）的建设，企业希望能够解决上述问题，实现仓储管理的智能化、自动化和高效化，从而提升整体生产运营效率。

### 1.1.2 **需求分析**

**原因分析**：离散制造行业的多品种小批量生产模式导致物料种类繁多、生产计划频繁变动，传统的仓储管理方式难以满足企业对库存精准管理、物料及时配送和仓储空间高效利用的需求。

**本质需求**：企业需要一个能够实时监控库存状态、精准追踪物料流向、智能规划仓储布局并实现物料自动配送的仓储管理系统，以提高仓储管理效率，降低库存成本，保障生产顺利进行。

**信息化系统价值**：WMS 系统可以通过与生产管理系统（MES）、供应协同系统（SCM）等的集成，实现从原材料采购到成品出库的全流程信息化管理，提高生产协同效率，优化库存水平，为企业决策提供数据支持。

**行业内友商解决方案**：如佰思杰科技的 WMS 系统，通过 RFID、自动化设备等先进科技手段，提升供应链物流作业效率及可视化管控程度，实现管理规范化、精细化、数字化及智能化。

### 1.1.3 **AI场景应用**

#### 1.1.3.1 **智能化库存管理**

**需求预测与优化**：AI 可以通过机器学习算法对历史数据进行分析，精准预测未来的库存需求，优化库存策略，减少库存积压和缺货风险。例如，系统可以根据销售数据、季节变化等因素预测未来一段时间内某种商品的需求量，提前安排备货，确保供应链的稳定运行。

**实时库存监控与自动补货**：AI 系统能够实时监控库存状态，自动识别库存异动，及时发现和调整库存短缺或过剩情况，并根据库存数据进行自动补货建议，避免人工判断失误。

#### 1.1.3.2 **智能化物料追踪**

**实时位置与状态监控**：通过 AI 技术，可以实时监控物料在仓库内的位置、数量、状态等信息，实现物料的全程追溯。例如，利用计算机视觉和传感器技术，系统可以实时跟踪物料的移动和存储情况，提高追踪的准确性和及时性。

**质量问题快速定位与处理**：当物料出现质量问题时，AI 系统可以迅速定位问题物料的来源、批次以及去向，及时采取纠正措施，减少对生产和供应链的影响。

#### 1.1.3.3 **数据分析与预测**

**运营数据可视化**：AI 智能仓储系统能够收集和分析大量的仓储数据，例如库存数据、订单数据、物流数据等。通过数据可视化技术，将这些数据以图表等直观的形式呈现出来，帮助管理人员快速了解仓储运营状况，及时发现问题和瓶颈。

**智能决策支持**：基于数据分析，AI 系统可以提供智能决策支持，例如优化库存管理策略、调整生产计划、改进物流调度等。例如，DeepSeek 的 AI 技术可以通过对数据的深入挖掘，发现潜在的规律和趋势，为企业提供精准的预测和决策建议。

**入库管理阶段**

**仓库管理员**：

**原料调拨入库**：根据原料调拨入库申请单，进行原料的调拨入库操作。

**余料入库**：根据余料入库申请单，进行余料的入库操作。

**成品完工入库**：根据成品完工入库申请单，进行成品的入库操作。

**盘点差异入库**：根据盘点差异入库申请单，进行盘点差异的入库操作。

**生成入库单**：完成原料调拨入库后，生成入库单，更新物料库存信息。

**系统**：

**物料库存**：更新物料库存信息，确保库存数据的实时性和准确性。在入库过程中，进行单位换算，确保库存数据的准确性。

**库存管理阶段**

**仓库管理员**：

**期初库存导入**：导入初始期初库存数据【未来扩展】。

**移位**：根据生产需求或库存管理需要，进行物料的移位操作。

**盘点**：定期进行库存盘点，生成盘点清单，确保库存数据的准确性。

**系统**：

**库存预警**：根据库存水平，生成库存预警信息，提醒仓库管理员进行补货或调整库存。

**物料库存**：实时更新物料库存信息，确保库存数据的准确性和完整性。

**系统**：

**物料库存**：更新物料库存信息，确保库存数据的实时性和准确性。



#### 2.1.1.2 **业务流程活动详情**

**入库管理阶段**

**涉及角色**：仓库管理员

**输入业务对象**：原料调拨入库申请单、余料入库申请单、成品完工入库申请单、盘点差异入库申请单

**输出业务对象**：入库单、物料库存

**关键业务规则**：

原料调拨入库、余料入库、成品完工入库和盘点差异入库均需生成相应的入库单。

入库单生成后，系统自动更新物料库存信息。

**业务规则配置点**：

库房库位分配规则



**库存管理阶段**

**涉及角色**：仓库管理员

**输入业务对象**：盘点清单、移位申请单

**输出业务对象**：物料库存、库存预警

**关键业务规则**：

定期进行库存盘点，生成盘点清单，确保库存数据的准确性。

按整个工厂来锁库存，盘点时不允许做物料的出入库操作（本次实现）

按库存、库位或物料锁库存，盘点时不允许做物料的出入库操作（未来扩展）

根据盘点结果，进行库存调整，生成相应的入库或出库单。

根据库存水平，生成库存预警信息，提醒仓库管理员进行补货或调整库存。

**业务规则配置点**：

盘点周期和方法的设置

库存预警的阈值设置



**出库管理阶段**

**涉及角色**：生产部门、仓库管理员

**输入业务对象**：领料出库申请单

**输出业务对象**：领料出库单、物料库存

**关键业务规则**：

生产部门根据生产需求提交领料出库申请单。

仓库管理员根据领料出库申请单进行物料拣配出库操作，生成领料出库单。

领料出库单生成后，系统自动更新物料库存信息。

**业务规则配置点**：

出库优先级规则



库存调整阶段

**涉及角色**：仓库管理员

**输入业务对象**：原料过期退库申请单、原料报废出库申请单、成品调拨出库申请单

**输出业务对象**：物料库存

**关键业务规则**：

原料过期退库、原料报废出库和成品调拨出库均需生成相应的出库单。

出库单生成后，系统自动更新物料库存信息。

**输入**：

物料的库存数量增加，具体计算方式为：物料的库存数量 = 物料的库存数量 + 本次入库数量。

物料库存唯一性：工厂 + 物料 + 库房状态 + 批次号+ 序列号 + 库房 + 库位。

入库申请单的已入库数量更新：已入库数量 = 原已入库数量 + 本次入库数量。

更新入库申请单的入库状态：

当计划数量 = 已入库数量，入库状态为全部入库。

当已入库数量 > 0，且已入库数量 < 计划数量，入库状态为部分入库。

当已入库数量 = 0，入库状态为未入库。

- **行内操作**: 在主数据网格的每一行操作列，始终显示`[按钮] 行内拣配`。

**业务规则**:
- **库存查找**: 拣配时，系统需根据出库申请单指定的物料（编码、版本）查找可用库存。
- **自动拣配优先级**: 系统需遵循业务配置中的出库优先级规则。
  - 读取业务配置`【仓储配置-物料出库-物料出库优先级规则】`。优先按库房配置读取，若无，则按组织配置读取。
  - **先进先出 (FIFO)**: 优先拣配入库时间最早的库存。
  - **有效期临近先出**: 优先拣配有效期最早到期的库存。
- **拣配数量**: 自动拣配的数量不能超过申请的"待出库数量"。

**处理逻辑**:
1. **触发事件**:
   - 用户在主数据网格勾选一个或多个申请单，点击`[按钮] 批量拣配`。
   - 或，用户点击某一行申请单的`[按钮] 行内拣配`。
2. **主流程**:
   1. 系统收集所有选中的出库申请单及其待出库的物料清单。
   2. 对清单中的每一种物料，系统根据"业务规则"中定义的优先级，在库存中查找最匹配的库存批次/序列号。
   3. 系统自动计算本次可拣配数量（取"申请待出库数量"和"可用库存数量"的最小值）。
   4. 将成功匹配并计算好数量的物料项，逐条添加到**拣货篮**中。
   5. 更新拣货篮顶部的汇总数据（总项数、总数量）。
   6. 主数据网格中对应申请行的"已拣配数量"和"拣配状态/进度"需要实时更新。
3. **后置条件**:
   - 物料成功添加到拣货篮。
   - 主列表的拣配状态和数量已更新。
4. **异常处理**:
   - 如果某个物料找不到任何可用库存，系统应给出明确提示（如："物料XXX库存不足，无法完成拣配"），并中止该物料的拣配操作，但不影响其他物料的拣配。

**验收标准**:
- 点击批量拣配或行内拣配后，符合规则的物料被成功添加到拣货篮。
- 添加到拣货篮的物料，其出库数量、库房、库位、批次号等信息均正确无误。
- 拣配后，主列表的"已拣配数量"和"拣配状态"更新正确。
- 库存不足时，系统能给出清晰的提示信息。


### 3.5.3 物料出库-从库存直接拣配
**概述**：允许用户在没有出库申请单的情况下，直接从现有库存中选择物料进行拣配，并将其加入拣货篮。这适用于补料、急料等临时出库场景。

**界面描述**:
- 在主工作区的操作栏，提供一个`[按钮] 从库存拣配`。
- 点击后，弹出一个"选择物料库存"的模态框（Modal）。
- **模态框内部**:
  - **查询区**: 提供库房、库位、库存状态、物料编码/名称等查询条件。
  - **库存列表**: 展示查询到的库存信息，包含物料、批次、序列号、库存数量等。
  - **操作**: 用户可选择一行或多行库存，并手动输入本次要拣配的数量。

**业务规则**:
- **拣配数量约束**: 用户手动输入的拣配数量不能超过所选库存行的"可用库存数量"。
- **出库方式**: 通过此方式拣配的物料，在拣货篮中其"出库方式"应标记为"直接出库"或类似的业务术语，以区别于"按申请出库"。

**处理逻辑**:
1. **触发事件**: 用户点击`[按钮] 从库存拣配`。
2. **主流程**:
   1. 系统弹出"选择物料库存"模态框。
   2. 用户使用查询条件找到目标库存，勾选并填写拣配数量。
   3. 用户点击模态框内的`[按钮] 确定`。
   4. 系统校验用户输入的数量是否合法（不大于可用库存）。
   5. 校验通过后，将用户选择的物料项添加到**拣货篮**。
   6. 更新拣货篮顶部的汇总数据。
3. **后置条件**:
   - 物料成功添加到拣货篮，并标记为"直接出库"。
4. **异常处理**:
   - 如果用户输入的拣配数量超过可用库存，系统应在模态框内给出错误提示，并阻止其添加到拣货篮。

**验收标准**:
- 能够成功从库存列表选择物料并指定数量加入拣货篮。
- 加入拣货篮的物料，"出库方式"标记正确。

### 3.5.5 物料出库-全部出库
**概述**：用户在拣货篮中完成所有物料的拣配确认后，可执行"全部出库"操作，系统将一次性处理所有物料的库存扣减，并生成正式的出库单。

**界面描述**:
- 在拣货篮的头部区域，提供一个`[按钮] 全部出库`。该按钮通常是拣货流程的最终操作。

**业务规则**:
- **库存扣减**: 确认出库时，系统需对拣货篮中的每一项物料，在数据库中执行真实的库存扣减操作。
- **负库存校验**:
  - 读取业务配置`【仓储配置-物料库存-是否允许负库存】`。
  - 如果不允许负库存，在执行扣减前必须校验操作是否会导致库存变为负数。若会，则中止出库并给出明确提示。
- **出库单生成**:
  - 出库成功后，系统需为本次出库操作生成一张或多张正式的出库单。
  - 出库单号需遵循编码规则，如 `CK+YYYYMMDD+NNNNN`。
- **状态更新**:
  - 更新相关出库申请单的"已出库数量"和"出库状态"（如更新为"部分出库"或"全部出库"）。

**处理逻辑**:
1. **触发事件**: 用户点击拣货篮中的`[按钮] 全部出库`。
2. **主流程**:
   1. 系统弹出二次确认对话框："您确定要将拣货篮中X项，共Y件物料执行出库吗？"。
   2. 用户确认后，系统开始处理。
   3. **事务开始**:
   4. 对拣货篮中的每一项物料，执行负库存校验（如果配置要求）。
   5. 执行库存数量的扣减。
   6. 生成出库单及出库单明细。
   7. 更新源头出库申请单的状态。
   8. **事务提交**。
   9. 清空拣货篮。
3. **后置条件**:
   - 物料库存已正确扣减。
   - 生成了合法的出库单。
   - 拣货篮被清空，主界面刷新。
4. **异常处理**:
   - 如果在处理过程中（如负库存校验失败），事务应回滚，所有数据恢复到出库前状态。
   - 系统需向用户显示清晰的失败原因，例如："物料XXX库存不足，出库失败，所有操作已取消。"
   - **特殊校验**: 当出库工厂下存在盘点状态为"待盘点"的盘点清单时，不允许出库，并提示用户："当前工厂正在盘点中，请盘点完成后再进行出库操作！"

**验收标准**:
- 点击全部出库后，拣货篮中所有物料的库存被正确扣减。
- 系统能成功生成结构正确、数据无误的出库单。
- 相关的出库申请单状态已正确更新。
- 负库存校验和盘点校验能按预期工作，并在不满足条件时阻止出库。

**展现形式**：查询结果以表格形式展示，包括物料编码、物料名称、物料版本、物料图号、库存状态、库存数量、计量单位、工厂、库房编码、库房名称、库位编码、库位名称、批次号、序列号、型号、项目号、有效期、安全库存、密级等信息

**处理逻辑**：

读取业务配置【仓储配置-物料预警-物料呆滞预警时长】

优先读根据库房读取配置，若库房为空，则根据组织读取配置

读不到则默认预警1年

根据配置的提前期，当物料入库后超过提前期未进行任何出入库（不包含库存盘点）时，则生成呆滞物料预警。

用户可以通过查询条件筛选出需要关注的呆滞物料。

系统将查询结果以表格形式展示在页面上。

**验收标准**：

**输入校验**：工厂为必填项，其他查询条件可为空。

**功能校验**：点击搜索按钮后，系统正确显示呆滞物料预警信息；点击重置按钮后，系统清空所有查询条件；点击展开按钮后，系统显示更多查询条件。

**界面校验**：查询条件和功能按钮显示正确，查询结果表格展示完整，数据正确。
